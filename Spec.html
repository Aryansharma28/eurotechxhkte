CareBridge 康橋

One-line: a transitional-care system that keeps recently-discharged elderly Hong Kong patients well at home — so they don't get readmitted. Three roles, one shared care picture, fed by a frictionless daily phone call to the elder.

01 Why this exists

Roughly 1 in 5 elderly patients are readmitted within 30 days of discharge, and most are preventable — missed medication, poor hydration, an untreated infection, a fall, fluid overload in heart-failure patients. The window that matters is the first ~30 days at home.

CareBridge closes that window by making daily monitoring effortless for everyone:

  • The elder does nothing but answer a phone — no app, no login, no screen.
  • The family gets quiet reassurance and only-when-it-matters nudges.
  • The social worker sees five elders at a glance and acts on flags before they escalate.

Pilot scope: Hong Kong, one social worker × five elders, bilingual 繁體中文 (Cantonese) + English.

03 Core concepts

Daily call 每日電話

Scheduled Cantonese voice check-in (AI-voice or staff). ~5 simple questions → structured activity records + flags. The primary data source.

ADL / daily activity

Six tracked items: Medication 服藥, Meals 進食, Mobility 活動, Hydration 飲水, Sleep 睡眠, Mood 情緒. Daily status: done / missed / pending.

Risk tier 風險級別

stable 穩定 (green) · watch 留意 (amber) · risk 高危 (red). Drives sort order & alerts. Derived, overridable.

Flag · Visit · Care plan

Flags = clinical signals from a call/note/visit. Visits = logged home checks. Care plan = clinician-set tasks per elder.

Derivations

  • riskTier: start stable; → watch on an open watch flag, ≥2 missed activities in 7d, or an out-of-range vital trend; → risk on any risk flag (e.g. CHF weight gain), a missed call while already on watch, or worker override.
  • Caseload is sorted by risk (risk → watch → stable).
  • Family "today" status is a friendly reduction of today's call + open flags — honest but calm when a flag is open.

05 The daily care loop

Scheduled daily call ─▶ Elder answers, speaks ─▶ Answers parsed into Activities + Flags
   ─▶ DailyCall.summary generated ─▶ Dashboard + Family app update live
   ─▶ Risk recomputed ─▶ if flag/risk: Alert on worker's Today screen
   ─▶ Worker triages: call elder/family, or schedule a home visit
   ─▶ Visit logged ─▶ loop continues until elder "graduates" (~Day 30)

Missed-call escalation: 2 retries; if still unanswered raise a 高危 risk flag and notify worker + family (see Chan Kwok-keung in the prototype). Graduation: around Day 30, stable elders are discharged from the programme (see Ho Yuen-wah).

06 Screens to build

Social worker — web dashboard (primary)

  • Today — greeting + KPI strip; "Needs your attention" alerts; caseload grid of 5 cards (risk pill, call status, 6 activity glyphs, top flag); today's visits; programme pulse.
  • Elder detail (drawer) — risk banner with "why"; this-week activity calendar (6×7); latest vitals; check-ins timeline; care plan; family contact.
  • Tablet visit mode — full-screen, large targets: tap-check activities, notes, photo, quick flags, Save & sync.

Adult child — mobile app

Single reassuring Today screen: status hero ("She's okay today" + call time), one gentle note if a flag is open, plain-language call summary, today's 6 activities, this week's check-ins, "How you can help today", sticky Call / Message actions. Bilingual toggle.

Elder — no UI

Just the call. Build the voice agent: scheduling, Cantonese prompts, retry logic, answer→data mapping, transcript + bilingual summary. Elder Call.html demonstrates the script and mapping.

07 Bilingual, accessibility & tone

  • Every string ships English + 繁體中文 (HK Cantonese register); language toggle on every surface, persisted per user. Call is Cantonese-first; transcripts stored bilingually.
  • Elder = accessibility by removal — no screen at all; warm, slow, one question at a time.
  • Family app — ≥15px body, large hero, 44px+ targets, plain language, never alarmist.
  • Dashboard — colour never the only signal (icon + label too); amber/red reserved strictly for triage. Honest reassurance: when something's wrong, say so plainly and say it's handled.

08 Suggested stack

  • Frontend: React + TypeScript; map styles/system.css tokens → Tailwind/CSS vars. Family app as PWA or React Native.
  • Backend: Postgres (schema above) + REST/tRPC; websockets for live dashboard updates.
  • Voice: telephony + Cantonese TTS/STT voice-agent for the call; LLM to parse answers → Activity/Flag and write the bilingual summary.
  • Compliance: health data — plan PDPO (HK) consent, per-role access control, and an audit log on every elder-record view.

09 Build phases (HK pilot)

M1
Data + dashboard read path. Schema, seed 5 elders, Today + caseload + elder detail (read-only).
M2
The call. Voice agent, scheduling, answer→data mapping, summaries → live Activities/Flags.
M3
Worker actions. Alerts, visit mode (write), care plan, risk recompute.
M4
Family app. Status hero, summary, nudges, contact.
M5
Pilot hardening. PDPO consent, audit, bilingual QA with real Cantonese speakers, escalation runbook.
